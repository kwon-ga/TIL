프레임워크 vs 라이브러리
모듈 vs 패키지

프레임워크란 ?
원하는 기능 구현에 집중하여 개발할 수 있도록 일정한 형태와 필요한 기능을 갖추고 있는 골격, 뼈대를 의미한다.
 어플리케이션 개발 시 필수적인 코드, 알고리즘, db연동과 같은 기능들을 위해 어느정도의 구조를 제공하며 이러한 뼈대 위에서 사용자는 코드를 작성하여 애플리케이션을 개발한다.
 앱/서버 등의 구동, 메모리 관리, 이벤트 루프 등의 공통된 부분은 프레임워크가 관리하며, 사용자는 프레임워크가 정해준 방식대로 클래서, 메서드를 구현한다.

예시)
java - Spring
python - django, flask
안드로이드 - android
애플 - cocoa touch
웹 - vue.js
등등

라이브러리란 ?
소프트에어를 개발할 때 컴퓨터 프로그램이 사용하는 비휘발성 자원의 모임. 즉 특정 기능을 모아둔 코드, 함수들의 집합이며 코드 작성 시 활용 가능한 도구들을 의미한다.

예시)
python - bs4, pandas
node.js - npm으로 설치한 모듈
html - jquery
웹개발 - react.js

리엑트는 리엑트 오피셜 라이브러리고
express는 node.js에서 가장 인기가 많은 web framework

라이브러리를 사용하여 개발을 하면 프레임워크가 그 코드를 사용한다.


모듈이란?
각종 변수, 함수, 클래스를 담고있는 파일.

패키지란?
여러 모듈을 묶은 것.
